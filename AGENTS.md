# AGENTS.md

## Стек проекта

- Приложение: React 19 + TypeScript, SPA на Vite.
- Роутинг: `react-router-dom` с маршрутами `/` и `/checkout`.
- Стили: Tailwind CSS 4 через `@tailwindcss/postcss`, глобальные стили в `src/styles/global.css`.
- Иконки: `lucide-react`.
- Данные: статический каталог товаров и категории в `src/data/products.ts`, валюты в `src/data/currencies.ts`.
- Логика UI: состояние корзины, валюты и фильтра категории хранится в `src/App.tsx`; страницы находятся в `src/pages`, переиспользуемые блоки в `src/components`.
- Сборка: Vite, TypeScript в строгом режиме (`strict: true`, `noEmit: true`, `moduleResolution: Bundler`).

## Ключевые команды

- `npm install` - установить зависимости из `package-lock.json`.
- `npm run dev` - запустить dev-сервер Vite на `127.0.0.1`.
- `npm run build` - выполнить проверку TypeScript (`tsc -b`) и production-сборку Vite в `dist`.
- `npm run preview` - открыть preview production-сборки на `127.0.0.1`.

В проекте сейчас нет отдельных npm-скриптов для линтинга, форматирования или тестов.

## Правила кода

- Пиши новый код на TypeScript и сохраняй строгую типизацию. Не добавляй `any`, если можно выразить тип через существующие модели из `src/types/shop.ts` или локальные типы props.
- Следуй текущей структуре:
  - компоненты UI - в `src/components`;
  - страницы маршрутов - в `src/pages`;
  - хуки - в `src/hooks`;
  - чистая бизнес-логика и форматтеры - в `src/logic`;
  - статические данные - в `src/data`;
  - общие типы - в `src/types`.
- Для компонентов используй именованные экспорты, как в текущих файлах (`export function Header(...)`).
- Состояние корзины и выбранной валюты держи централизованно в `App.tsx`, если изменение касается нескольких страниц или компонентов.
- Для визуальных правок сначала ищи решение в Tailwind-классах. Глобальный CSS меняй только для базовых правил, фокуса, сбросов и действительно общих стилей.
- Поддерживай текущий визуальный язык: темный фон, zinc/emerald-палитра, `rounded-md`/`rounded-lg`, компактные карточки, адаптивные сетки.
- Для интерактивных элементов сохраняй доступность: кнопкам задавай `type="button"` вне submit-форм, кликабельным карточкам оставляй `role`, `tabIndex` и обработку клавиш, у изображений должны быть осмысленные `alt`.
- Не добавляй новые runtime-зависимости без явной необходимости. Если нужна иконка, сначала используй `lucide-react`, который уже есть в проекте.
- В исходниках есть строки с поврежденной кириллической кодировкой рядом с нормальной кириллицей. Не выполняй массовую перекодировку или замену текстов без отдельной задачи; при локальной правке проверяй, что пользовательский текст отображается ожидаемо.
- Не коммить и не правь артефакты сборки (`dist`, `*.tsbuildinfo`), если задача прямо не требует обновить build output.

## Инструкции самопроверки

Перед сдачей изменений:

1. Запусти `npm run build`.
2. Если менялась верстка или поведение UI, запусти `npm run dev` и проверь в браузере:
   - главную страницу `/`;
   - фильтрацию категорий;
   - открытие модального окна товара;
   - добавление товара в корзину, изменение количества и удаление;
   - переход на `/checkout`;
   - состояние пустой корзины и успешное оформление демо-заказа.
3. Проверь адаптивность минимум на мобильной ширине и desktop: карточки, корзина, hero-блок и checkout не должны накладываться друг на друга.
4. Если менялись цены, валюты или корзина, проверь форматирование через `formatPrice` и пересчет итоговой суммы.
5. Если менялись countdown или хуки, проверь поведение после обновления страницы и отсутствие лишних интервалов/таймеров.
6. Убедись, что `git status --short` показывает только ожидаемые файлы.
